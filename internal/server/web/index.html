Each folder is either a full path or a path relative to the vault root above — both work. Folders/scan-roots override the server's env defaults. Blank Book fields fall back to the matching Light Novel field, so a single-folder setup keeps working unchanged.

Reading

The completed-reads log — full path or relative to the vault root. Marking a volume/book completed appends a row here; leave blank to disable reading tracking.

Schedule

Standard 5-field cron expressions, checked independently on their own schedules. "Run check" (above) always runs both in one pass. Saving takes effect immediately, no restart needed.

Matching

Import from Calibre

idle

One-time-ish bulk import of a Calibre library into the vault. Books are matched against OpenLibrary / jnovels / Lubimyczytać and staged as review notes in the staging folder below; you review them in Obsidian, then Finalize to move the keepers into the vault. Nothing is written into your scan roots until you finalize.

Paths

The staging folder is kept out of your scan roots so half-reviewed notes are never tracked. Leave it as _CalibreImport unless you have a reason to change it.

Filter (optional)

Import only books carrying a particular Calibre identifier (the type:value pairs on a book, e.g. czyj:andrzej). Leave the field blank to import everything.

Multiple owners: since Calibre stores one value per identifier, a value like andrzej; czyj:arek (comma- or semicolon-separated) is read as two owners and matches either — but andrzej arek (space) is one value and matches neither.

Dry-run preview

A cheap, network-free scope check — counts what a run would cover, so a wrong library path is caught before anything is matched or written.

Run

Matching a large library takes a while and is polite-throttled against each source. Stop halts after the current item; you can Resume later and it picks up where it left off, reusing matches it already found.

Finalize

After you've reviewed the staged notes in Obsidian — deleting the duplicates/rejects you don't want — Finalize moves the survivors into the vault. Non-destructive: an existing note is never overwritten (a collision is skipped and reported).
